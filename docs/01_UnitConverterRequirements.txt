1. 개요 (Overview)
================================================================================

1.1 시스템 목적
    - 사용자가 입력한 길이(형식: 단위:값)를 기반으로, 해당 값을 다른 모든
      지원 단위로 변환하여 출력하는 프로그램을 제공한다.

1.2 설계 목표
    - 새로운 단위를 추가할 때 기존 코드의 변경이 최소화되도록 설계한다.
    - 각 단위 변환 로직은 테스트 코드로 검증한다.

2. 기능 요구사항 (Functional Requirements)
================================================================================

2.1 사용자 입력 및 출력
--------------------------------------------------------------------------------
요구사항 ID : FR-001
요구사항명  : 단위:값 형식 입력
설명        : 사용자는 "단위:숫자" 형식으로 길이 값을 입력할 수 있어야 한다.
입력 예시   : meter:2.5
우선순위    : 필수

요구사항 ID : FR-002
요구사항명  : 전체 단위 변환 결과 출력
설명        : 입력된 단위와 값에 대해, 지원하는 모든 단위로 변환한 결과를
              출력해야 한다.
출력 예시   :
              2.5 meter = 8.2 feet
              2.5 meter = 2.7 yard
              ...
우선순위    : 필수

2.2 지원 단위
--------------------------------------------------------------------------------
요구사항 ID : FR-003
요구사항명  : 기본 지원 단위
설명        : 시스템은 최소한 다음 길이 단위를 지원해야 한다.
              - meter (미터)
              - feet (피트)
              - yard (야드)
우선순위    : 필수

요구사항 ID : FR-004
요구사항명  : 단위 확장 시 변경 최소화
설명        : 새로운 단위가 추가될 때 기존 코드의 변경이 최소화되도록
              설계·구현해야 한다. (Open-Closed Principle 준수)
우선순위    : 필수

2.3 테스트
--------------------------------------------------------------------------------
요구사항 ID : FR-005
요구사항명  : 단위 변환 정확성 검증
설명        : 각 단위 간 변환이 정확히 계산되도록 테스트 코드를 작성해야 한다.
우선순위    : 필수

3. 비즈니스 로직 (Business Rules)
================================================================================

요구사항 ID : BR-001
규칙명      : meter ↔ feet 변환 비율
내용        : 1 meter = 3.28084 feet

요구사항 ID : BR-002
규칙명      : meter ↔ yard 변환 비율
내용        : 1 meter = 1.09361 yard

요구사항 ID : BR-003
규칙명      : feet ↔ yard 간 변환
내용        : feet와 yard 간의 변환 비율은 meter를 기준 단위로 하여 계산한다.
              (모든 변환은 meter 기준으로 일관되게 수행)

================================================================================
4. 품질 요구사항 (Quality Requirements)
================================================================================

요구사항 ID : QR-001
요구사항명  : OCP(Open-Closed Principle) 준수
설명        : 확장(새 단위·비율·출력 형식 추가)에는 열려 있고,
              기존 모듈 수정에는 닫혀 있는 설계를 만족해야 한다.
우선순위    : 필수

요구사항 ID : QR-002
요구사항명  : SRP(Single Responsibility Principle) 준수
설명        : 클래스는 단일 책임을 갖도록 구성한다.
              (예: 입력 파싱, 검증, 변환 계산, 출력 포맷 등 역할 분리)
우선순위    : 필수

요구사항 ID : QR-003
요구사항명  : 입력 값 검증
설명        : 다음 경우에 대한 입력 검증을 구현해야 한다.
              - 음수 값
              - 잘못된 입력 형식 (단위:값 형식 위반 등)
              - 존재하지 않는(미등록) 단위
우선순위    : 필수

5. 추가 요구사항 (Additional Requirements)
================================================================================

5.1 설정 외부화
--------------------------------------------------------------------------------
요구사항 ID : AR-001
요구사항명  : 변환 비율 외부 설정 로드
설명        : 변환 비율을 외부 설정 파일(JSON 또는 YAML)에서 로드할 수
              있어야 한다.
우선순위    : 추가

5.2 동적 단위 등록
--------------------------------------------------------------------------------
요구사항 ID : AR-002
요구사항명  : 런타임 단위·비율 등록
설명        : 사용자 입력을 통해 단위와 비율을 동적으로 등록하고,
              등록 직후 변환에 사용할 수 있어야 한다.
등록 예시   : 1 cubit = 0.4572 meter
우선순위    : 추가

5.3 출력 포맷 선택
--------------------------------------------------------------------------------
요구사항 ID : AR-003
요구사항명  : 다중 출력 포맷 지원
설명        : 변환 결과를 다음 형식 중 선택하여 출력할 수 있어야 한다.
              - JSON
              - CSV
              - 표(테이블) 형태
우선순위    : 추가
